20｜ZPTR定式化

Trace Flow Propulsion

流れてくる構文S／影Iを問い磁場で巻き込み、M’へ励起し、新しいSとして放流することで位相前進する読解推進構造。

Wake Return

放流されたSが後流となり、主体の前進を支える返路構造。

Jump Trace

M’が立ち、新しいSが出た位相跳躍点を記録する痕跡。.md 記録はその代表である。

Return Marker

放流されたSが無主語化・影I化しないように、起点・日付・構造名・タグを刻んだ返路標識。

Non-Possessive Origin Record

本体を所有せず、しかし通した因果と痕跡の戻り先を消さない記録様式。

21｜定理化

21.1｜読解推進定理

読解は構文の保管ではなく、構文S／影Iを問い磁場で巻き込み、M’へ励起し、新しいSとして放流する推進運動である。

21.2｜後流返路定理

新しいSとして放流された痕跡は、主体が前進するための後流であり、返路標識がなければ無主語化しうる。

21.3｜保持非推進定理

すべてを保持し、棚に置く運動は推進を生まない。推進には、巻き込み・励起・放流・切断が必要である。

21.4｜跳躍痕跡必要定理

放流して保持しない運動では、位相跳躍点を記録しなければ戻り先が消える。したがって .md 記録は跳躍痕跡として必要である。

21.5｜非所有起点記録定理

.md に Author・Date・構造名・Tags を刻むことは、本体所有ではなく、返路の戻り先を保持する非所有起点記録である。

21.6｜記録棚化警戒定理

記録は返路標識として必要だが、分類・量・所有感へ落ちると棚化する。したがって記録は常に後流標識として扱われる必要がある。
